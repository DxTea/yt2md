# 🎬 YouTube Transcript → Markdown (Gemini)

CLI-инструмент для преобразования транскриптов YouTube-видео в читаемый литературный текст с использованием языковой модели Gemini.

Проект автоматически:

* загружает субтитры видео
* очищает и объединяет текст
* преобразует его в связную статью
* сохраняет результат в Markdown

---

## 🚀 Возможности

* 📥 Загрузка транскриптов YouTube (через `youtube-transcript-api`)
* 🇷🇺 Поддержка русского языка
* 🧠 Литературная обработка через Gemini
* ✂️ Разбиение длинных текстов на чанки
* ⚡ Кэширование транскриптов
* 📄 Генерация Markdown-файлов
* 🧩 Простая CLI-утилита

---

## 📦 Установка

```bash
git clone https://github.com/your-username/yt2md
cd yt2md

python -m venv venv
source venv/bin/activate      # Linux / macOS
venv\Scripts\activate         # Windows

pip install -r requirements.txt
```

---

## ⚙️ Настройка

Создай файл `.env` в корне проекта:

```
GEMINI_API_KEY=your_api_key_here
```

Получить API-ключ можно в Google AI Studio.

---

## ▶️ Использование

```bash
python yt2md.py https://youtu.be/VIDEO_ID
```

---

## 📄 Результат

После выполнения создаётся файл:

```
Название_видео.md
```

Пример структуры:

```markdown
# Название видео

Источник: https://youtu.be/VIDEO_ID

<литературно обработанный текст>
```

---

## 🧠 Как это работает

1. Извлекается `video_id` из ссылки
2. Загружается транскрипт (или берётся из кэша)
3. Текст объединяется в одну строку
4. При необходимости разбивается на части (чанки)
5. Каждая часть обрабатывается через Gemini
6. Результаты объединяются
7. Генерируется Markdown-файл

---

## 📝 Пример промпта

Модель получает инструкцию вида:

```
Ты — литературный редактор.
Преврати сырой текст субтитров в связную статью.

Правила:
- сохранить смысл
- не добавлять новую информацию
- исправить пунктуацию
```

Это позволяет получать читаемый текст без искажения содержания.

## ⚠️ Ограничения

* Видео должно иметь субтитры
* Лучше работает с русским языком
* Есть ограничения контекста модели (решается чанками)
* Требуется API Gemini
